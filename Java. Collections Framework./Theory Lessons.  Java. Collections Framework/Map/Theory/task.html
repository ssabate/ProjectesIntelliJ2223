interface Map<K,V>

K is the type of keys and V is the type of stored values
The java.util.Map interface represents a mapping between a key and a value. The Map interface is not a subtype of the Collection interface. A map cannot contain duplicate keys; each key can map to at most one value.

Map Implementations:

  • java.util.HashMap
  • java.util.Hashtable
  • java.util.EnumMap
  • java.util.IdentityHashMap
  • java.util.LinkedHashMap
  • java.util.Properties
  • java.util.TreeMap
  • java.util.WeakHashMap

*the most commonly used Map implementations are HashMap and TreeMap.

hierarchy:



*The hierarchy of classes very similar to the hierarchy of Sets

HashMap maps a key and a value. It does not guarantee any order of the elements stored internally in the map.

TreeMap also maps a key and a value. Furthermore it guarantees the order in which keys or values are iterated - which is the sort order of the keys or values. Check out the JavaDoc for more details.

LinkedHashMap - extends HashMap class. It creates a linked list of elements in the map, located in the order in which they were inserted.

in order to iterate through the map you need to know:

Set<Map.Entry<K, V>> entrySet() – Returns a map view as a set of Map objects.Entry, i.e. pairs "key-value". From this view you can delete items, they are removed from the map, but not add them.
Set<K> keySet() – Returns a map view as a set of all keys. From this view you can remove elements, while the keys and associated values are automatically removed from the map, but adding new elements is impossible.
Collection<V> values() – Returns the map view in the form of the set of all values. From this view, you can remove items, the values and corresponding keys are automatically removed from the map, but adding new elements is impossible.

some useful methods:

﻿default V putIfAbsent(K key, V value) - If the specified key is not already associated with a value (or is mapped ﻿to null) associates it with the given value and returns null, else returns the current value

default V getOrDefault(Object key, V defaultValue) - Returns the value to which the specified key is mapped, or defaultValue if this map contains no mapping for the key.

default V computeIfAbsent(K key, Function<? super K,? extends V> mappingFunction) - If the specified key is not already associated with a value (or is mapped to null), attempts to compute its value using the given mapping function and enters it into this map unless null.

default V computeIfPresent(K key, BiFunction<? super K,? super V,? extends V> remappingFunction) - If the value for the specified key is present and non-null, attempts to compute a new mapping given the key and its current mapped value.

boolean containsKey(Object key) - Returns true if this map contains a mapping for the specified key.
*there is a similar method - containsValue(Object value)
